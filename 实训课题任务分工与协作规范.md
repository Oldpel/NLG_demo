# 基于轻量化预训练模型与深度学习的中文文本情感分析系统 —— 任务分工与协作规范

> 本文档用于指导多人团队高效协作完成本科深度学习实训课题研究及报告书写。
> 适用团队规模：2~4人。

---

## 1. 项目概述

本课题旨在构建一个完整的中文文本情感分析系统，覆盖数据工程、模型训练、系统部署与报告撰写全生命周期。系统包含两条技术路线：

- **传统深度学习路径**：BiLSTM-Attention
- **轻量化预训练模型路径**：DistilBERT 微调

最终通过 Gradio 实现可视化交互界面，完成从数据到服务的工程闭环。

---

| **M3 预训练模型** | DistilBERT 微调与调优 | 第4.2~4.3节 | `distilbert_model.py`, `train_distilbert.py`, 指标JSON | M4 |

## 3. 各模块详细分工与接口契约

### 3.1 M1 数据工程组

#### 职责清单
- 下载 ChnSentiCorp 数据集，进行探索性数据分析（EDA：样本分布、文本长度分布）。
- 实现清洗管线：`regex` 去HTML/URL → `OpenCC` 繁简转换 → 去停用词（仅BiLSTM路径需要）。
- 生成标准化数据文件：`train.csv`, `val.csv`, `test.csv`（必须包含 `text`, `label` 两列）。
- 构建词表（上限30,000）和 Jieba 分词脚本。
- 为 DistilBERT 路径准备好 HuggingFace `Dataset` 对象或统一的 `CustomDataset` 类。

#### 输出规范（接口契约）

`dataset.py` 必须暴露以下接口，供 M2/M3 调用：

```python
from torch.utils.data import Dataset

class SentimentDataset(Dataset):
    def __init__(self, csv_path: str, model_type: str = "bilstm", max_len: int = 128):
        """
        model_type: "bilstm" 或 "distilbert"
        """
        ...

    def __getitem__(self, idx: int) -> dict:
        """
        返回内容约定：
        - bilstm 路径: {"input_ids": LongTensor(seq_len), "label": int}
        - bert 路径:   {"input_ids": LongTensor(128), "attention_mask": LongTensor(128), "label": int}
        """
        ...
```

#### 交付物检查清单
- [ ] `data/raw/` 原始数据备份
- [ ] `data/cleaned/train.csv`, `val.csv`, `test.csv`
- [ ] `data/vocab.json`（词表，30k上限）
- [ ] `dataset.py`（双路径兼容）
- [ ] 数据清洗前后对比统计（用于报告）

### 3.2 M2 传统模型组（BiLSTM-Attention）

#### 职责清单
- 实现 `Embedding + BiLSTM + SelfAttention + MLP` 网络结构。
- 支持加载外部预训练词向量（Tencent/FastText），也可随机初始化。
- 训练脚本需支持：CPU/GPU 自适应、早停（Early Stopping）、模型 checkpoint 保存。
- 独立评估脚本：输出 `Accuracy`, `Precision`, `Recall`, `F1` 到 JSON 文件。

#### 输出规范（接口契约）

模型文件 `models/bilstm/model.py` 必须实现：

```python
import torch.nn as nn

class BiLSTMAttentionClassifier(nn.Module):
    def __init__(self, vocab_size: int, embed_dim: int, hidden_dim: int, num_classes: int = 2):
        ...

# 推理接口（M4会直接调用这个签名）
def predict(text: str, model, tokenizer, device: str = "cpu") -> dict:
    """
    返回统一格式:
    {"label": "正向积极评论" | "负向消极评论", "score": float, "prob": float}
    例如: {"label": "正向积极评论", "score": 0.9856, "prob": 0.9856}
    """
    ...
```

- [ ] `models/bilstm/model.py`
- [ ] `train_bilstm.py`（含早停、checkpoint保存）
- [ ] `evaluate_bilstm.py`（输出指标JSON）
- [ ] `checkpoints/bilstm_best.pth`
- [ ] 训练曲线截图（Loss & Accuracy）

### 3.3 M3 预训练模型组（DistilBERT）

#### 职责清单
- 基于 `transformers` 加载中文 DistilBERT（如 `distilbert-base-chinese` 或同类模型）。
- 实现微调脚本：AdamW 优化器、Linear Warmup + Cosine Annealing、可选 Layer-wise Learning Rate Decay。
- 严格监控显存占用，确保在 **6GB 显存**下 `batch_size=32`, `max_len=128` 可运行。
- 与 M2 保持一致的评估指标输出格式，方便最终做对比表。

#### 输出规范（接口契约）

模型文件 `models/distilbert/model.py` 必须实现与 M2 完全一致的推理接口：

```python
from transformers import AutoModelForSequenceClassification, AutoTokenizer

# 推理接口（必须与M2完全一致）
def predict(text: str, model, tokenizer, device: str = "cpu") -> dict:
    """
    返回统一格式:
    {"label": "正向积极评论" | "负向消极评论", "score": float, "prob": float}
    """
    ...
```

#### 交付物检查清单
- [ ] `models/distilbert/model.py`

- [ ] `evaluate_distilbert.py`（输出指标JSON，格式同M2）

### 3.4 M4 系统集成组（全栈/Engineering）

#### 职责清单
- **模型工厂（Model Factory）**：自动根据配置加载 M2 或 M3 的权重，暴露统一预测接口。
- **输入清洗与安全防护**：对前端输入做正则过滤（防注入、防异常字符），不要信任前端输入。
- **Gradio 前端**：
  - 面板A：单文本输入 + 实时情感标签 + 置信度柱状图（`gr.Label`）。
  - 面板B：CSV 拖拽上传 → 批量推理 → 结果下载。
- **压力测试**：边界输入（空字符串、纯标点、超长文本、代码片段）测试系统鲁棒性。
- **延迟测试**：记录端到端推理耗时（目标 30~45ms）。

#### 关键接口设计

`api.py` —— 整个项目的"胶水层"：

```python

class SentimentAnalyzer:
    def __init__(self, model_type: str = "distilbert", model_path: str = "./checkpoints/..."):
        """
        model_type: "bilstm" | "distilbert"
        """

    def predict(self, text: str) -> Dict[str, float | str]:
        """单条文本推理"""
        ...

    def predict_batch(self, texts: List[str]) -> List[Dict[str, float | str]]:
        """批量文本推理，用于CSV处理"""
        ...
```

## 4. 协作流程与 Git 规范

### 4.1 分支策略（Git Flow 简化版）

```
main          # 稳定分支，只合并经过联调验证的代码
  ├── dev     # 日常开发集成
  ├── feat/m1-data-pipeline
  ├── feat/m2-bilstm
  ├── feat/m3-distilbert
  └── feat/m4-gradio-ui
```

### 4.2 协作约定
- **M1/M2/M3 可以并行启动**，但 M1 必须在第 3 天前合并到 `dev`（输出标准化数据格式）。
- **M4 在 `dev` 分支上工作**，等 M2/M3 的 `predict` 接口稳定后做一次 **联调日（Integration Day）**。
- 所有模型权重文件（`.bin`, `.pth`）**不要提交 Git**，使用网盘/服务器共享；代码仓库中只保留 `checkpoints/.gitkeep`。
- 每次合并到 `dev` 前必须保证脚本可运行（至少在自己的环境中通过冒烟测试）。

## 6. 风险管理与兜底方案

| 风险场景 | 应对方案 | 负责人 |
| :--- | :--- | :--- |
| **显存不够**（DistilBERT OOM） | 立即降级：减小 `batch_size` 到 16 或 8，增大梯度累积步数（`gradient_accumulation_steps`） | M3 |
| **Jieba 分词效果差** | 切换为 `pkuseg` 或 LTP 作为备选分词器，词表构建逻辑不变 | M1 |
| **Gradio 渲染卡顿** | 将同步推理改为 `gr.Button` 触发的异步队列，或加 `gr.Progress` 进度条 | M4 |
| **某成员进度落后** | M4 提前写好 `mock_predict()` 假接口（返回随机概率），保证前端可先独立开发 | M4 |
| **模型训练不收敛** | 检查学习率是否过大、数据标签是否错乱、loss函数是否正确；必要时先在小样本上过拟合验证 pipeline 正确性 | M2/M3 |
